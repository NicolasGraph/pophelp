Preferences: Allow more Textile markup in comments?

When set to ‘Yes’, comments may contain Textile (opens an external link in a new window) elements which would otherwise be treated as plain text, such as:

  • headings
  • ordered and unordered lists
  • footnotes
  • tables
  • code
  • extended block syntax (bc.., bq.., etc.)

Note: For security reasons, raw HTML is transformed into regular text.
