<field name="varname">acronym</field>
<field name="area">textile</field>
<field name="lang">es-es</field>
<field name="title">Definir acrónimos</field>
<field name="data_raw">HTML (Lenguaje de Marcas de Hipertexto) permite a los autores definir acrónimos vía la etiqueta &lt;acronym&gt;. La definition aparece como un tool tip cuando el cursor se sitúa sobre el acrónimo. Esta ayuda crucial para facilitar la lectura, debería ser utilizada al menos una vez para cada acrónimo en los documentos donde aparezcan.

Para definir rápidamente un acrónimo en Textile, coloca el texto completo entre (paréntesis) inmediatamente después del acrónimo.

Ejemplo:

bq. ==ACLU(American Civil Liberties Union)==

Se convertirá en:

bq. ==&lt;acronym title=&quot;American Civil Liberties Union&quot;&gt;ACLU&lt;/acronym&gt;==

Que, cuando se muestre en un navegador, tendrá un aspecto parecido al siguiente (mueve tu cursor sobre el texto):

bq. ACLU(American Civil Liberties Union)

&lt;small&gt;NB: Textile agrupa automáticamente una cadena de 3 o más letras mayúsculas en la etiqueta &lt;acronym&gt;, incluso si no tiene definición.&lt;/small&gt;

&quot;Más información(más información sobre esta etiqueta en diveintoaccessibility.org)&quot;:http://diveintoaccessibility.org/day_17_defining_acronyms.html</field>
</row>

<field name="data_raw">Para encerrar un párrafo completo en un blockquote, coloca &quot;bq. &quot; delante.

bq. Un texto antiguo.

bq. bq. Una marca de bloque.

bq. Cualquier texto antiguo.

Se convertirá:

bq. &lt;p&gt;Un texto antiguo.&lt;/p&gt;

bq. &lt;blockquote&gt;Una marca de bloque.&lt;/blockquote&gt;

bq. &lt;p&gt;Cualquier texto antiguo.&lt;/p&gt;

Que, cuando sea renderizado por un navegador, se mostrará como esto:

Un texto antiguo.

bq. Una marca de bloque.

Cualquier texto antiguo.

<row>
<field name="varname">css_mode</field>
<field name="area">prefs</field>
<field name="lang">es-es</field>
<field name="title">Modo CSS</field>
<field name="data_raw">El estándar de &quot;Hojas de Estilo en Cascada&quot;:http://www.w3schools.com/css/default.asp es un sencillo y potente lenguaje que te permite centralizar el control de la presentación del contenido de toda tu web. Todo, de fuentes a colores, pasando por tamaños y posiciones puede gobernarse con CSS.

Ordinariamente los datos CSS aparecen en la sección &lt;code&gt;head&lt;/code&gt; de un documento HTML, o se almacenan en un fichero de texto enlazado al documento. Los navegadores modernos parsean estos datos antes de mostrar una página, y guardan en caché estos datos para emplearlos en las páginas siguientes o en posteriores visitas. De este modo, el empleo de CSS puede reducir ampliamente el tamaño de las páginas a descargar: sólo uno de los muchos beneficios que ofrece el formateo con CSS sobre los antiguos métodos de diseño web.

Textpattern ofrece dos modos de mantenimiento para las hojas de estilo de tus páginas.

En Modo CSS Manual, puedes crear (o pegar) cualquier información de estilos que quieras, simplemente en la interfaz de Textpattern. Esto te permitirá editar rápidamente estilos sin la tediosa tarea de subir archivos por ftp. Las hojas de estilos pueden  haber sido generadas desde cero o con un editor WYSIWYG(What You See Is What You Get - Lo que ves es lo que tienes), y que pueden encontrarse en &quot;todo sobre la  web&quot;:http://directory.google.com/Top/Computers/Programming/Internet/CSS/Sample_Style_Sheets/Layouts/.

En Modo CSS Automático, la edición de hojas de estilo se lleva a un nuevo nivel de sofisticación, usando una interfaz de edición y un método de organización que pretenden convertir los parámetros de CSS en algo más legible y lógico. Cualquier hoja de estilos existente puede ser 'echada' en la interfaz de edición y modificada indefinidamente.

En Modo Automático, los archivos CSSson clasificados tanto por layout como por  _disposición_ como por _estilo_. Las páginas de una web pueden combinar o compartir cualquier tipo.

Los archivos de disposición (Layout) definen parámetros principalmente estructurales, tales como el númeo y el tamaño de columnas en una página, cabeceras y pies, &amp;c.

Por su parte, los archivos de Estilo (Style) se encargan de cosas como tipografía, color, y los métodos más finos de posicionamiento (e.g. padding y márgenes y las relaciones entre los objetos de una página) no definidas en los ficheros de disposición.</field>
</row>

Cada vez que un usuario hace clic en el enlace permanente de un artículo (por ejemplo http://estesitio/archivo/123/), el artículo se muestra con la plantilla de página asignada a esta sección. Para ir a una página con la lista de los archivos más recientes asignados a esta categoría, la URL tendría que ser http://estesitio/archivo/.

*acerca de*: una página 'acerca de', que es diferente de la página de inicio y la de archivo, en el sentido de que es más estática, puede tener una plantilla de página diferente y, presumiblemente sólo tendrá un artículo. La sección 'acerca de' se encontrará en http://estesitio/acercade/.

*fotos*: una sección con una plantilla de página distinta (y tal vez una hoja de estilo distinta) para mostrar fotos en vez de artículos.

Ten en cuenta que siempre existe una _sección_ especial, la sección llamada *default*. La página de inicio de tu sitio se construye usando la plantilla de página llamada 'default' y la hoja de estilo 'default'.</field>
</row>

<field name="title">Usando secciones</field>
<field name="data_raw">Las secciones en Texpattern son importantes en el sentido de que no sólamente indican una clase a la que pertenecen los artículos. Cada sección tiene asignado una plantilla de página (creada por ti o generada por Texpattern), un estilo en forma de archivo CSS (Cascading Style Sheet) y una ubicación específica en el servidor.

Cuando publicamos un artículo nuevo, toda esta información relativa a la forma en que se visualizará el artículo, se determina asignando una Sección al artículo.

Dicho esto, la primera sección que debemos fijar es una que no puede ser asignar de ninguna manera a los artículos, denominada *default*. 

Para un sitio simple puede ser suficiente una única sección (llamada 'articulos', por ejemplo).</field>
</row>

<field name="data_raw">Hace poco se descubrió una importante vulnerabilidad de PHP. &quot;La alerta de seguridad dice&quot;:http://www.hardened-php.net/advisory_202005.79.html says:

bq. puede conducir a un comportamiento inesperado de las aplicaciones PHP, lo que puede conducir a la ejecución de código remoto PHP en varias situaciones.

Las versiones vulnerables son: de PHP4 &lt;= 4.4.0, y de PHP5 &lt;= 5.0.5. Deberías, o bien actualizar a versiones más recientes, o asegurarte de desactivar register_globals.

Puedes desactivar register_globals en el archivo php.ini. Si estás en un servicio de alojamiento, tal vez tengas una opción en el panel de control para hacerlo (pregunta a tu servidor). Cuando se usa mod_php (php como un módulo de Apache) también es posible desactivar register_globals por medio de .htaccess. Descomenta la siguiente línea de tu .htaccess:

@php_value register_globals off@

<field name="data_raw">Esta preferencia debería dejarse en blanco mientras no surjan problemas al mandar emails desde TXP (o sea: que no mande ningún email). <field name="varname">forms_overview</field>
<field name="area">form</field>
<field name="lang">es-es</field>
<field name="title">Bloques</field>
<field name="data_raw">En Textpattern, un Bloque (en inglés &lt;em&gt;Form&lt;/em&gt; = Formulario) difiere de su significado usual de _serie de campos para rellenar_, como en la expresión Formulario de Contacto o Formulario de Búsqueda. En lugar de eso, es una pieza de texto, de HTML o de etiquetas de Textpattern que se puede reutilizar para mostrar contenido.

Aquí hay un ejemplo de Bloque que se podría llamar @ordinary_list@:

bc. &lt;li&gt;
   &lt;txp:permlink&gt;
      &lt;txp:title /&gt;
   &lt;/txp:permlink&gt;
&lt;/li&gt;

Puedes usar este bloque (&lt;em&gt;form&lt;/em&gt; en inglés) para mostrar una lista no numerada HTML de artículos relacionados:

bc. &lt;txp:related_articles form=&quot;ordinary_list&quot; wraptag=&quot;ul&quot; /&gt;

Como la función de este bloque es mostrar una lista de títulos, se puede usar dentro de otras etiquetas, por ejemplo para mostrar 20 artículos de la sección actual:

bc. &lt;txp:article form=&quot;ordinary_list&quot; limit=&quot;20&quot; wraptag=&quot;ol&quot; /&gt;

También puedes usar los Bloques para mostrar contenido repetido, como el encabezamiento y el pie en las plantilas de Página. Esto significa que el contenido sólo está en un sitio y que, si decides cambiarlo, sólo tienes que hacerlo una vez. Para mostrar contenido de un Bloque, normalmente se hace con:

bc. &lt;txp:output_form form=&quot;your_form_name&quot; /&gt;

Los Bloques pueden ser tan largos como quieras. Por ejemplo el Bloque para un pie de página puede incluir encabezamientos, párragos, enlaces, etiquetas condicionales y otros bloques como la @ordinary_list@.</field>
</row>
